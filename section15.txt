5. What Happens to the User's Files Elsewhere?
A critical point to understand: userdel -r only removes the user's home directory and mail spool. If the user owned files anywhere else on the system (e.g., in /tmp/, /var/www/, or a shared directory), those files will remain on the system. Their ownership will change to the former user's UID (a "ghost" UID). System administrators often need to find and reassign or delete these files manually using tools like find.

# Example: Find files owned by UID 1001 (max's old UID) anywhere on the system
sudo find / -uid 1001 2>/dev/null
